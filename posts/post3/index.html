Bài viết 3 - Khám phá lập trình hướng đối tượng trong Java (OOP)

September 17, 2025 4-minute read
Java • Cisco Networking Academy • OOP • Lập Trình Hướng Đối Tượng • Java Essentials 1

“Bài viết này, mình chính thức bước vào thế giới của OOP – nơi mọi thứ trong lập trình bắt đầu trở nên có tổ chức và thực tế hơn.”

🎯 Mục tiêu Link to heading

  • Hiểu rõ khái niệm lớp (class) và đối tượng (object).
  • Nắm được constructor, method, và tính đóng gói (encapsulation).
  • Thực hành viết chương trình quản lý đơn giản bằng nhiều đối tượng.

💡 Cảm nhận đầu tiên Link to heading

Ban đầu, mình hơi “choáng” với hàng loạt khái niệm mới: class, object, method, constructor, encapsulation…
Tuy nhiên, Cisco thiết kế bài học rất hợp lý — họ bắt đầu bằng ví dụ đời thường, như “một chiếc xe là một đối tượng có thuộc tính (màu, tốc độ) và hành vi (chạy, dừng)”.
Nhờ vậy, mình nhanh chóng hiểu được:
👉 Class giống như bản thiết kế, còn Object là bản thể cụ thể được tạo ra từ bản thiết kế đó.

📚 Kiến thức trọng tâm Link to heading

🔹 1. Lớp (class) và đối tượng (object) Link to heading

Mình học cách định nghĩa một lớp trong Java.
Ví dụ: lớp Student có hai thuộc tính và một phương thức.

public class Student {
    String name;
    int age;

    void showInfo() {
        System.out.println("Tên: " + name);
        System.out.println("Tuổi: " + age);
    }
}

Tạo đối tượng từ lớp này:

Student s1 = new Student();
s1.name = "Minh";
s1.age = 20;
s1.showInfo();

Khi chạy, màn hình hiển thị thông tin sinh viên – giúp mình hiểu rõ hơn sự khác biệt giữa class và object.

🔹 2. Hàm tạo (constructor) Link to heading

Constructor là “phương thức đặc biệt” dùng để khởi tạo đối tượng.

public class Student {
    String name;
    int age;

    Student(String n, int a) {
        name = n;
        age = a;
    }

    void showInfo() {
        System.out.println(name + " - " + age);
    }
}

Student s2 = new Student("An", 19);
s2.showInfo();

Nhờ constructor, mình có thể khởi tạo nhanh dữ liệu ban đầu mà không cần gán từng dòng.

🔹 3. Tính đóng gói (Encapsulation) Link to heading

Cisco dạy rằng tính đóng gói giúp ẩn chi tiết bên trong đối tượng và chỉ cho phép truy cập qua các phương thức getter/setter.

public class Account {
    private double balance;

    public void setBalance(double b) {
        if (b > 0)
            balance = b;
    }

    public double getBalance() {
        return balance;
    }
}

Ở đây, biến balance bị “che giấu”, chỉ truy cập được thông qua hai hàm get và set.
Điều này giúp bảo vệ dữ liệu khỏi việc bị sửa sai cách.

🔹 4. Phương thức và phạm vi truy cập Link to heading

Mình học về các modifier như public, private, protected và hiểu rằng chúng quyết định phạm vi truy cập trong chương trình.
Ví dụ: public có thể gọi ở bất kỳ đâu, còn private chỉ dùng được trong class.

📝 Kinh nghiệm học tập Link to heading

  • 🧩 Vẽ sơ đồ lớp: Tập vẽ Class Diagram đơn giản (tên lớp, thuộc tính, phương thức) giúp dễ hình dung quan hệ giữa các lớp.
  • 🧠 Chạy từng phần nhỏ: Viết – chạy – sửa dần từng hàm, không viết cả chương trình một lần.
  • 🖥️ In log bằng console: Dùng System.out.println() để theo dõi tiến trình code.
  • ✍️ Đặt tên rõ nghĩa: Đặt tên lớp, hàm, biến sao cho dễ đọc, ví dụ calculateAverage() thay vì ca().

⚙️ Khó khăn & Cách khắc phục Link to heading

🧩 Hiểu sai phạm vi biến (scope) Link to heading

Nhiều lần mình khai báo biến trong hàm rồi lại dùng bên ngoài → lỗi “cannot find symbol”.
Giờ mình nhớ rõ: biến chỉ tồn tại trong cặp {} của nó.

🧱 Quên this trong constructor Link to heading

Khi tên tham số trùng với tên thuộc tính, cần viết this.name = name; để phân biệt.

🤔 Nhớ nhầm giữa class và object Link to heading

Mình tự nhủ: “class là khuôn mẫu – object là sản phẩm.”
Ví dụ: “Student” là bản thiết kế, còn Minh là một đối tượng cụ thể của lớp đó.
