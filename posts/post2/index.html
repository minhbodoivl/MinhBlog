Bài viết 2 – Bước đầu vào thế giới lập trình hướng đối tượng

September 10, 2025 4-minute read
Java • Cisco Networking Academy • Lập Trình Hướng Đối Tượng • Java Essentials 1

“Bài viết thứ hai, mình chính thức bước vào thế giới lập trình, nơi những dòng code đầu tiên bắt đầu có ý nghĩa.”

🎯 Mục tiêu Link to heading

  • Làm quen với Java Essentials 1.
  • Hiểu cú pháp cơ bản: biến, kiểu dữ liệu, cấu trúc chương trình.
  • Thực hành viết chương trình, vòng lặp, và điều kiện.

💡 Cảm nhận đầu tiên Link to heading

Sau Bài viết đầu tiên làm quen với Networking Basics, mình chính thức bước sang giai đoạn thứ hai của khoá học tại Cisco Networking Academy – đó là Java Essentials 1.
Đây cũng là Bài viết đầu tiên mình thực sự code nhiều, và bắt đầu cảm nhận “hơi thở” của ngôn ngữ lập trình Java.

Trước đây mình đã nghe nhiều về Java – một ngôn ngữ mạnh mẽ, phổ biến, dùng từ ứng dụng desktop đến Android. Nhưng khi bắt đầu học chính thức, điều đầu tiên mình cảm thấy là Java chặt chẽ và có quy tắc rõ ràng.
Mọi thứ từ cách đặt tên lớp, dấu ngoặc nhọn, đến khai báo biến đều phải chính xác.

Điểm mình thích là môi trường học của Cisco có IDE trực tuyến, cho phép viết – biên dịch – chạy chương trình ngay trên trình duyệt, không cần cài đặt.
Điều này giúp mình tập trung vào việc học ngôn ngữ hơn là cài công cụ.

📚 Kiến thức học được Link to heading

🔸 Cấu trúc chương trình Java Link to heading

Một chương trình Java luôn bắt đầu bằng class chứa hàm main(), ví dụ:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Xin chào thế giới Java!");
    }
}

💬 Mình hiểu được rằng mọi thứ trong Java đều phải nằm trong một lớp, và main() là điểm bắt đầu của chương trình.

🔸 Biến và kiểu dữ liệu Link to heading

Mình được học cách khai báo và sử dụng các kiểu dữ liệu cơ bản:

int tuoi = 20;
double diemTB = 8.5;
char kyTu = 'A';
boolean daTotNghiep = false;
String ten = "Minh Coder";

Cảm giác khá dễ hiểu, vì cú pháp Java rõ ràng, gần giống C#, nhưng có quy tắc nghiêm ngặt hơn.

🔸 Toán tử và điều kiện Link to heading

Các bài tập giúp mình ôn lại kiến thức logic như if, else, switch, toán tử so sánh (==, !=, >, <) và toán tử logic (&&, ||). Ví dụ bài tập tính điểm trung bình và phân loại học sinh:

if (diemTB >= 8.0)
    System.out.println("Giỏi");
else if (diemTB >= 6.5)
    System.out.println("Khá");
else
    System.out.println("Trung bình");

🔸 Vòng lặp trong Java Link to heading

Mình luyện tập viết các vòng lặp for, while, do-while để tính tổng, in bảng cửu chương, hoặc kiểm tra số nguyên tố.

Scanner sc = new Scanner(System.in);
System.out.print("Nhập n: ");
int n = sc.nextInt();

int sum = 0;
for (int i = 1; i <= n; i++) {
    sum += i;
}

System.out.println("Tổng từ 1 đến " + n + " là: " + sum);

📝 Kinh nghiệm học tập Link to heading

  • ⏰ Chia nhỏ mục tiêu mỗi ngày: chỉ học 1–2 module, tránh học dồn vì Java cần thời gian để “ngấm”.

  • ✍️ Tự gõ code, không copy: giúp nhớ cú pháp lâu hơn và phát hiện lỗi dễ hơn.

  • 💬 Dùng comment để hiểu code: tập thói quen thêm chú thích // hoặc /* ... */.

  • 💡 Thực hành thêm ngoài bài học: sau mỗi module, mình tự nghĩ thêm 1–2 bài tập nhỏ như:

    • Viết chương trình đổi nhiệt độ C ↔ F.
    • Tính tổng các số lẻ từ 1 đến 100.
    • In tam giác sao theo chiều cao nhập vào.

⚙️ Khó khăn & Cách vượt qua Link to heading

🧩 Lỗi cú pháp liên tục

  • Vấn đề: chỉ thiếu dấu {} hoặc ; là lỗi.
  • Giải pháp: học cách đọc lỗi terminal để sửa nhanh.

🧠 Nhớ quy tắc đặt tên

  • Vấn đề: lớp viết hoa chữ cái đầu (StudentInfo), biến viết thường (studentAge).
  • Giải pháp: lặp lại quy tắc mỗi khi viết class mới.

💭 Khái niệm “class” và “object”

  • Vấn đề: còn khá trừu tượng.
  • Giải pháp: chuẩn bị tinh thần để hiểu kỹ hơn trong Bài viết 3, khi học lập trình hướng đối tượng (OOP).
