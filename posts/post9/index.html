Bài viết 9 - Đa hình (Polymorphism) trong Java

📚 Kiến thức trọng tâm Link to heading

🔹 1. Khái niệm đa hình (Polymorphism) Link to heading

“Polymorphism” nghĩa là nhiều hình dạng khác nhau.
Trong Java, nó xảy ra khi một đối tượng của lớp con được tham chiếu bởi lớp cha.

class Animal {
    void sound() {
        System.out.println("Động vật phát ra âm thanh");
    }
}

class Dog extends Animal {
    void sound() {
        System.out.println("Gâu gâu!");
    }
}

class Cat extends Animal {
    void sound() {
        System.out.println("Meo meo!");
    }
}

Animal a;
a = new Dog();
a.sound(); // Gâu gâu!
a = new Cat();
a.sound(); // Meo meo!

Mặc dù a có kiểu khai báo là Animal, nhưng hành vi thực tế lại phụ thuộc vào đối tượng thật (Dog hay Cat).

🔹 2. Compile-time vs Runtime Polymorphism Link to heading

  • Compile-time (Static): thường là method overloading – cùng tên, khác tham số.
  • Runtime (Dynamic): là method overriding – lớp con định nghĩa lại hành vi của lớp cha.
// Compile-time
class MathUtils {
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }
}

// Runtime
class Shape {
    void draw() { System.out.println("Vẽ hình..."); }
}

class Circle extends Shape {
    void draw() { System.out.println("Vẽ hình tròn!"); }
}

Shape s = new Circle();
s.draw(); // Gọi phương thức của lớp Circle (runtime polymorphism)

🔹 3. Upcasting và Downcasting Link to heading

Upcasting: Lớp con → lớp cha (tự động).
Downcasting: Lớp cha → lớp con (phải ép kiểu thủ công).

Animal a = new Dog();  // Upcasting
a.sound();             // Gâu gâu!

Dog d = (Dog) a;       // Downcasting
d.sound();

⚠️ Nếu downcast sai kiểu, sẽ lỗi ClassCastException.

🔹 4. Lớp trừu tượng (Abstract class) Link to heading

Một lớp abstract có thể chứa phương thức trừu tượng (abstract method) – chưa có nội dung, bắt buộc lớp con phải ghi đè.

abstract class Shape {
    abstract void draw();
}

class Rectangle extends Shape {
    void draw() {
        System.out.println("Vẽ hình chữ nhật!");
    }
}

Shape s = new Rectangle();
s.draw(); // Vẽ hình chữ nhật!

Abstract class rất hữu ích khi muốn định nghĩa “khuôn mẫu” cho các lớp con.

🔹 5. Interface và đa hình Link to heading

Interface định nghĩa các hành vi mà lớp phải thực hiện, cho phép đa kế thừa hành vi.

interface Animal {
    void sound();
}

class Dog implements Animal {
    public void sound() {
        System.out.println("Gâu gâu!");
    }
}

Animal dog = new Dog();
dog.sound();

Interface thể hiện đa hình ở cấp độ hành vi – nhiều lớp có thể cùng “thực hiện” một interface theo cách riêng.

🧠 Bài tập thực hành Link to heading

💬 Bài 1: Đa hình qua lớp cha Link to heading 

class Vehicle {
    void move() {
        System.out.println("Phương tiện đang di chuyển");
    }
}

class Bike extends Vehicle {
    void move() {
        System.out.println("Xe đạp đang chạy...");
    }
}

class Car extends Vehicle {
    void move() {
        System.out.println("Xe hơi đang lăn bánh...");
    }
}

Vehicle v;
v = new Bike();
v.move();
v = new Car();
v.move();

🧩 Bài 2: Abstract class Link to heading 

abstract class Employee {
    abstract void work();
}

class Developer extends Employee {
    void work() {
        System.out.println("Lập trình và fix bug");
    }
}

class Tester extends Employee {
    void work() {
        System.out.println("Kiểm thử phần mềm");
    }
}

Employee e1 = new Developer();
Employee e2 = new Tester();
e1.work();
e2.work();

🧱 Bài 3: Interface Link to heading 

interface Drawable {
    void draw();
}

class Circle implements Drawable {
    public void draw() {
        System.out.println("Vẽ hình tròn");
    }
}

class Square implements Drawable {
    public void draw() {
        System.out.println("Vẽ hình vuông");
    }
}

Drawable d1 = new Circle();
Drawable d2 = new Square();
d1.draw();
d2.draw();

📝 Kinh nghiệm học tập Link to heading

  • ⚙️ Luôn khai báo theo kiểu cha (hoặc interface), giúp chương trình dễ mở rộng.
  • 🧠 Hiểu rõ upcasting/downcasting, đặc biệt là khi dùng danh sách đối tượng hỗn hợp.
  • 💡 Interface giúp tách rời phần định nghĩa và phần thực thi → code linh hoạt hơn.
  • 🚀 Tư duy hướng hành vi (behavior) thay vì hướng lớp cụ thể.

⚙️ Khó khăn & Cách khắc phục Link to heading

😵 Nhầm giữa override và overload Link to heading

Mình từng nghĩ hai cái là một, nhưng thật ra:

  • Overload: cùng tên, khác tham số.
  • Override: cùng tên, cùng tham số, khác lớp (cha – con).

🧩 Ép kiểu sai Link to heading

Khi downcast sai kiểu (VD: ép Cat thành Dog), chương trình lỗi ngay.
Giờ mình luôn kiểm tra bằng instanceof trước khi ép kiểu.

🤔 Abstract vs Interface Link to heading

Ban đầu hơi mơ hồ, nhưng mình nhớ mẹo này:

“Abstract – là khung xương; Interface – là bản hợp đồng.”
