Bài viết 4 - Ôn tập điều kiện và vòng lặp trong Java

September 24, 2025 5-minute read
Java • Cisco Networking Academy • Điều Kiện • Vòng Lặp • Java Essentials 1

“Bài viết này, mình quay lại với những khối kiến thức nền tảng – điều kiện và vòng lặp.
Dù quen thuộc, nhưng khi học bằng Java, mọi thứ trở nên rõ ràng và chặt chẽ hơn.”

🎯 Mục tiêu Link to heading

  • Ôn tập câu lệnh rẽ nhánh (if, else if, switch).
  • Thành thạo các vòng lặp (for, while, do-while).
  • Thực hành với những bài toán nhỏ như tính tổng, in bảng cửu chương, kiểm tra số nguyên tố.
  • Hiểu rõ tư duy lặp và điều kiện dừng trong chương trình.

💡 Cảm nhận đầu tiên Link to heading

Bài viết này giống như quay lại “gốc rễ” của lập trình – những cấu trúc điều khiển mà ngôn ngữ nào cũng có.
Tuy nhiên, Java khiến mình thấy mọi thứ trở nên rõ ràng, chặt chẽ và có quy tắc hơn.
Mình đặc biệt thích cách Cisco thiết kế các ví dụ nhỏ: dễ hiểu, sát thực tế và giúp mình tự tay “cảm nhận” được vòng lặp hoạt động thế nào qua từng giá trị.

📚 Kiến thức trọng tâm Link to heading

🔹 1. Câu điều kiện (if, else if, switch) Link to heading

Dùng để xử lý các tình huống phân nhánh logic dựa trên điều kiện.

int score = 7;

if (score >= 8) {
    System.out.println("Giỏi");
} else if (score >= 5) {
    System.out.println("Trung bình");
} else {
    System.out.println("Yếu");
}

Ngoài if, mình còn học thêm switch-case – rất hữu ích khi có nhiều lựa chọn rõ ràng.

int day = 3;

switch (day) {
    case 1: System.out.println("Thứ Hai"); break;
    case 2: System.out.println("Thứ Ba"); break;
    case 3: System.out.println("Thứ Tư"); break;
    default: System.out.println("Ngày không hợp lệ");
}

🔹 2. Vòng lặp for Link to heading

Phù hợp khi biết trước số lần lặp.
Ví dụ: tính tổng các số từ 1 đến 10.

int sum = 0;

for (int i = 1; i <= 10; i++) {
    sum += i;
}

System.out.println("Tổng từ 1 đến 10 là: " + sum);

🔹 3. Vòng lặp while Link to heading

Dùng khi chưa biết trước số lần lặp, nhưng có điều kiện dừng.

int i = 1;

while (i <= 5) {
    System.out.println("Lần thứ " + i);
    i++;
}

🔹 4. Vòng lặp do...while Link to heading

Khác với while, vòng lặp này luôn chạy ít nhất một lần trước khi kiểm tra điều kiện.

int num;

do {
    System.out.print("Nhập số (0 để dừng): ");
    num = scanner.nextInt();
} while (num != 0);

🔹 5. Kết hợp điều kiện và vòng lặp Link to heading

Ví dụ: in tất cả số nguyên tố nhỏ hơn 100.

for (int i = 2; i < 100; i++) {
    boolean isPrime = true;
    for (int j = 2; j * j <= i; j++) {
        if (i % j == 0) {
            isPrime = false;
            break;
        }
    }
    if (isPrime) {
        System.out.print(i + " ");
    }
}

Những bài như thế này giúp mình rèn tư duy thuật toán, chứ không chỉ học cú pháp.

📝 Kinh nghiệm học tập Link to heading

  • 🧠 Luôn dự đoán trước khi chạy code: đoán xem vòng lặp sẽ chạy bao nhiêu lần, rồi so sánh với kết quả thật.
  • 🪜 Dùng biến đếm (i, j) hợp lý: tránh đặt tên mơ hồ, nên đặt theo ý nghĩa như index, count, num.
  • 🖥️ In log để kiểm tra: thêm System.out.println() trong vòng lặp để xem biến thay đổi ra sao.
  • 🧩 Tách nhỏ bài toán: ví dụ, khi kiểm tra số nguyên tố, tách riêng phần kiểm tra chia hết trước khi ghép vào vòng lặp chính.

⚙️ Khó khăn & Cách khắc phục Link to heading

🔄 Nhầm giữa while và do...while Link to heading

Ban đầu mình quên rằng do...while luôn chạy ít nhất một lần. Sau vài lần test, mình đã quen cách kiểm soát điều kiện dừng.

🧮 Sai phạm vi biến Link to heading

Mình từng đặt biến đếm trong vòng lặp nhưng lại dùng nó bên ngoài → lỗi “cannot find symbol”. Giờ mình luôn nhớ: biến trong khối {} chỉ tồn tại trong khối đó.

🧱 Quên reset giá trị Link to heading

Khi dùng biến sum hoặc count, đôi khi mình quên đặt lại giá trị ban đầu trước mỗi vòng lặp → kết quả sai. Mình khắc phục bằng cách luôn khởi tạo rõ ràng.

  • Làm quen với vòng lặp lồng nhau để duyệt mảng hai chiều.

🌱 Học vòng lặp giúp mình hiểu rằng lập trình là việc dạy máy tính “lặp lại” công việc con người làm — nhưng không biết mệt, không sai sót, và luôn chính xác. Khi hiểu vòng lặp, mình thấy mình thực sự bắt đầu “nói chuyện” được với máy tính.
